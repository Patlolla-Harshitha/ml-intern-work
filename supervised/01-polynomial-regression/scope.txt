2
Subtopics

A
Core concepts and intuition

Extending linear regression by adding polynomial feature terms to capture non-linear patterns while keeping the model linear in its parameters.

B
Mathematical formulation

Hypothesis ŷ = θ₀ + θ₁x + θ₂x² + … + θdx^d; closed-form normal equation; cost function J(θ) as MSE; feature matrix X̃ construction.

C
Algorithm variants and extensions

Ridge polynomial regression, Lasso polynomial regression, interaction terms, multivariate polynomial expansion, piecewise polynomials (splines), Chebyshev basis.

D
Implementation details

From-scratch feature matrix construction; scikit-learn Pipeline with PolynomialFeatures; standardization before fitting; numerical stability considerations for high d.

E
Failure modes, edge cases, and misconceptions

Runge's phenomenon, feature scale explosion, collinearity between x^k terms, confusing model complexity with expressiveness, forgetting to scale features.

3
Top 5 interview questions (FAANG-level)

Design
You have a dataset where residuals from linear regression show a clear curved pattern. Walk me through every decision you'd make before and after choosing polynomial regression.

Tests: diagnostic mindset, degree selection, validation strategy, regularization, when to stop.

Theory
Polynomial regression is still a linear model — explain precisely what "linear" refers to here and why this distinction matters for learning algorithms.

Tests: understanding linearity in parameters vs. inputs; connection to the general linear model framework.

Tradeoff
As polynomial degree d increases, describe quantitatively what happens to training error, test error, model variance, and model bias. What does the optimal d look like?

Tests: bias-variance tradeoff depth; ability to reason about the U-shaped test error curve.

Practical
A colleague fits a degree-10 polynomial and gets perfect training accuracy but terrible validation performance. They propose collecting more data — will this help? What else would you suggest?

Tests: diagnosing overfitting vs. data quantity; regularization, degree reduction, cross-validation, learning curves.

ML depth
How does polynomial regression relate to kernel methods, and why would you ever choose a kernel SVM over a polynomial regression for a non-linear problem?

Tests: feature space intuition, curse of dimensionality, computational cost of explicit vs. implicit feature maps.

5
Connections to other concepts

Leads from
Sibling / related
Leads to
Linear regression

Polynomial regression is a direct generalization — same normal equation, same loss function.

Feature engineering

Polynomial expansion is a manual feature engineering strategy — a gateway to learning why features matter.

Ridge & Lasso regression

Regularization is nearly always applied alongside polynomial regression; inseparable in practice.

Splines & GAMs

Piecewise polynomials (splines) address Runge's phenomenon that global polynomials suffer from.

Kernel methods (SVM, GPR)

The polynomial kernel implicitly computes polynomial feature maps — the bridge to kernelized learning.

Neural networks

Motivates why deep networks learn non-linear representations — poly regression shows the limits of manual feature maps.

Model selection & CV

Degree d is a hyperparameter — polynomial regression is the canonical example for teaching hyperparameter search.

Bias-variance theory

The clearest empirical illustration of the bias-variance tradeoff across all of supervised learning.
